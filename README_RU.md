Данный проект представляет собой программную реализацию криптографического протокола **Эвена — Гольдрайха — Лемпеля (EGL)** для схемы забывчивой передачи данных «1-из-2» на базе системы RSA.

---

## 💡 О чем этот проект?
Забывчивая передача (Oblivious Transfer) — это технология, позволяющая отправителю (Алисе) передать одно из нескольких сообщений получателю (Бобу) так, чтобы:
1. **Алиса** не узнала, какое именно сообщение выбрал Боб.
2. **Боб** узнал только выбранное сообщение и не получил никакой информации об остальных.

---

## 🔥 Ключевые возможности
✅ **Графический интерфейс (GUI):** Современное окно на Tkinter с интерактивным логированием каждого шага.
✅ **Глубокая аналитика (Метрики):**
   - ⏱️ **Время:** Оценка каждого этапа (Keygen, Blinding, Encryption) с точностью до миллисекунд.
   - 📊 **Трафик:** Детальный подсчет объема передаваемых данных в **байтах** для каждого сетевого пакета.
✅ **Безопасная математика:**
   - Генерация 1024-битных RSA-ключей.
   - Использование криптографически стойкого генератора случайных чисел (`secrets`).
   - Реализация теста простоты Миллера-Рабина.

🧬 Как это работает (Этапы протокола)

1.  Инициализация (Alice): Генерирует пару ключей RSA (n, e, d) и два случайных
    значения x_0, x_1.
2.  Ослепление (Bob): Выбирает сообщение b, генерирует случайный секрет k и
    отправляет Алисе значение V = (x_b + k^e) \pmod n.
3.  Двойное шифрование (Alice): Алиса вычисляет два потенциальных ключа
    расшифрования k_0, k_1 (не зная, какой из них верный для Боба) и
    маскирует оба своих сообщения: C_i = M_i + k_i \pmod n.
4.  Получение (Bob): Боб получает C_0, C_1 и снимает маску со своего сообщения,
    просто вычитая секрет k.

🖥️ Скриншоты работы



Программа выводит пошаговый лог, подсвечивая действия Алисы синим, а Боба —
зеленым. В конце каждой операции указан точный размер переданных данных в
байтах.
